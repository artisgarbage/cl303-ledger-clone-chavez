# Ticket: Issue #9 - Narrative UI polish + historical seed for codelab303

**Issue:** https://github.com/artisgarbage/cl303-ledger-clone-chavez/issues/9  
**Budget:** $20 USD  
**Estimated tokens:** ~150k tokens

## Goal

Polish the Reports page (`/reports`) by adding period/basis controls, meaningful error states, improved history display, custom query UX enhancements, and a seed script that pre-populates 7 canonical narratives for codelab303 so the platform launches with real AI-generated analysis.

## Current State Analysis

✅ **Working:**
- `POST /api/analysis/narrative` - supports all narrative types, comparison periods, basis
- `GET /api/analysis/narrative` - returns narrative list
- `/reports` page exists with basic generation capability
- `PeriodSelector` component exists with preset + basis toggle
- 4 `FinancialPeriod` rows exist for codelab303 (2024, 2025, 2026 YTD CASH/ACCRUAL)
- Reports link already in sidebar navigation

⚠️ **Needs Work:**
- Reports page hard-codes period to "current month" and basis to "CASH"
- No period preview or custom date range support
- Error states are silent (no user feedback on failure)
- History cards don't show the period covered
- Custom query has no guidance chips
- No seed narratives - fresh install has empty history

## Approach

### Phase 1: Period Controls (AC #1)
1. Import `PeriodSelector` types and `getPeriodRange` into reports page
2. Add state for `preset`, `basis`, `customStart`, `customEnd`
3. Replace hard-coded period logic with dynamic calculation from state
4. Add "Custom" option to preset selector (modify `PeriodSelector` or create local variant)
5. Add custom date inputs that appear when "Custom" is selected
6. Add period preview badge showing selected range + basis
7. Wire up the `generate()` function to use dynamic period/basis

### Phase 2: Error Handling (AC #2)
1. Add error state: `const [error, setError] = useState<string | null>(null)`
2. Update `generate()` to extract error from non-OK responses
3. Add error banner component with amber styling, AlertTriangle icon, dismissible
4. Add specific error messages for 503 (missing API key), 400 (no data), network failures
5. Clear error on type/period change

### Phase 3: History List Improvements (AC #3)
1. Add period display to each narrative card subtitle
2. Implement relative period labels for standard periods (Q1 2025, Full Year 2024, etc.)
3. Move generation timestamp to secondary position
4. Add empty state illustration/message when history is empty

### Phase 4: Custom Query UX (AC #4)
1. Add suggested query chips (4 examples)
2. Implement chip click to pre-fill textarea
3. Add character counter (500 max)
4. Disable Generate button when over limit

### Phase 5: Seed Narratives Script (AC #5)
1. Create `prisma/seed-narratives.ts`
2. Implement idempotent narrative generation for 7 canonical narratives
3. Add guards for missing ANTHROPIC_API_KEY
4. Add sequential generation with logging
5. Import and call from `prisma/seed.ts`

### Phase 6: Testing & Documentation
1. Test all period presets + custom range
2. Test error states (remove API key, invalid dates, etc.)
3. Run seed script with and without API key
4. Verify history display with various period types
5. Update `.cl303/learnings.md` with findings

## Acceptance Criteria

1. ✅ Period selector works with all presets + custom date range
2. ✅ Error banner shows for all error cases with specific messages
3. ✅ History cards display period + basis prominently
4. ✅ Custom query has 4 suggested chips + character counter
5. ✅ Seed script generates 7 narratives idempotently
6. ✅ Seed script handles missing API key gracefully
7. ✅ Reports visible in sidebar (already done)

## Files to Modify
