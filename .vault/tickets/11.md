## Approach

Given budget constraints ($30 remaining), I will prioritize the most critical fixes that can be completed within budget:

### Priority 1: SEC-03 - Make User.companyId non-null (P0)

**Why this is critical:**
- Orphaned users (companyId=null) bypass multi-tenant isolation
- Required for type-safe tenant scoping
- Schema-level enforcement prevents bugs

**Implementation:**
1. Check current DB for orphaned users
2. Create Prisma migration to make companyId non-null
3. Update auth.ts session callback to rely on non-null type
4. Add constraint test
5. Update requireTenant helper to remove null checks

**Estimated effort:** 1 PR, ~60 lines, ~$8

### Priority 2: SEC-04 - Add audit logging (P0)

**Why this is critical:**
- Compliance requirement for SOC 2 / GDPR
- Forensic capability for security incidents
- Customer trust requirement for financial platform

**Implementation:**
1. Add AccessAudit Prisma model
2. Create logAccess() helper
3. Instrument key read routes (narratives/[id], periods, people/[id])
4. Add unit tests

**Estimated effort:** 1 PR, ~120 lines, ~$12

### Priority 3: SEC-05 - Replace xlsx dependency (P1 → HIGH)

**Why elevated to P0:**
- HIGH severity Prototype Pollution CVE
- Attack vector via file upload
- No fix available from upstream

**Implementation:**
1. Replace xlsx with exceljs
2. Update QuickBooks parser
3. Regression test with real codelab303 P&L export

**Estimated effort:** 1 PR, ~80 lines, ~$10

### Deferred to follow-up (within budget but lower priority):

**SEC-06: prisma migrate deploy** - P1, can be fixed in follow-up issue since current docker-compose is dev-only
