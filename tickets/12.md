## Mission Summary

Build Margot Hale, an omni-present CFO agent that works across web chat and Slack. Margot is a conversational financial partner who analyzes the existing financial data model, generates narrative reports on demand, and frames financial reality for different audiences (internal leadership, prospective clients, board/investors).

### Critical Learnings from `.cl303/learnings.md`
1. **Next.js 16 quirk:** Route params are async - must use `{ params: Promise<{ id: string }> }` and `await params`
2. **Anthropic model:** Use full ID `claude-sonnet-4-20250514` not shorthand
3. **Multi-tenant isolation:** All routes must verify `companyId` and `userId` match session via `requireSession()`
4. **Financial accuracy:** All currency calculations use `Decimal` types, never `Float`
5. **Auth helpers:** `src/lib/auth-helpers.ts` exports `requireSession()`, `requireAdmin()`, `requireTenant()` with guaranteed non-null `companyId`
6. **Contractor lag & variable employee cost:** Key business logic in cost calculations
7. **AI egress:** Line-item names must be redacted (`[REDACTED]`) before sending to Anthropic
8. **Audit logging:** `logAccess()` helper in `src/lib/audit/logger.ts` for SOC 2/GDPR compliance
9. **Seed data:** `FinancialPeriod` rows exist with real QB data from seed-financials.ts

### Deviations from Default Stack
- **XLSX library:** Uses `exceljs` instead of `xlsx` (HIGH CVE remediation in SEC-05)
- **No Redis/queue service:** Issue explicitly forbids - use Postgres + Next.js only
- **No vector DB:** Issue explicitly forbids for v1 - use Postgres `tsvector` or LIKE for search

### Repository Conventions Observed
- **Migration naming:** `YYYYMMDD_descriptive_slug` (e.g., `20260511_cfo_agent`)
- **Route handlers:** Async params pattern for `[id]` routes, `requireSession()` for auth
- **Error responses:** Structured JSON with `{ error: "message" }`, appropriate status codes
- **Component structure:** Server components by default, client components use `"use client"` directive
- **Tool pattern:** Each tool has `name`, `description`, `input_schema` (Zod → JSON Schema), and handler function
- **Fire-and-forget logging:** Audit logging catches exceptions and logs to console, never blocks operations

## Hard Constraints (from Issue)

1. ✅ Never invent numbers (enforced by tool responses)
2. ✅ Never expose internal cost in proposal mode (guard enforcement, not model behavior)
3. ✅ Never write to financial source-of-truth tables from tools (all tools read-only)
4. Always cite period and basis for P&L numbers (guard auto-appends if missing) - **M3**
5. Tool latency budget: >8s emits heartbeat - **M2**
6. Token budget: <80k per turn, summarize after 30 turns - **M2**
7. ✅ No vector DB, Redis, queue service (Postgres + Next.js + Anthropic SDK only)
8. ✅ TypeScript strict, zero `any`
9. Tests for every tool, loop, guards, and dialogues - **M2-M5**
10. ✅ Honor `.vault/directives/` and `AGENTS.md`

## Out of Scope (Non-Goals)

- Voice / telephony / audio
- Auto-posting on a schedule (reactive only in v1)
- Multi-tenant Slack (one install per Company)
- Cross-conversation memory (per-conversation context only)
