## Goal

Build Margot Hale, a conversational CFO agent that:
- Operates across **two surfaces** (web chat + Slack bot) sharing one core
- Provides ad-hoc financial analysis, generates narrative reports, frames financials for different audiences
- Has three **mode lenses**: internal-cfo (default), proposal-bizdev (sanitized for clients), board-investor (formal)
- Uses existing analytical primitives (`cost-basis.ts`, `project-profitability.ts`, etc.) as tools
- Maintains conversation state with turn-by-turn persistence
- Streams responses on web, batched on Slack

## Approach - Phased Delivery

Following the issue's 5-phase breakdown (M1-M5), but adapted to realistic scope for this run:

### Phase M1 - Core Loop (Web Only) [This PR]

**Scope:** Foundational architecture, web chat surface, 3 basic tools, sync responses
**Acceptance:** Ask "what's our gross margin trend over the last three months" and get a cited answer

**Implementation:**

3. **API routes**:

   - `DELETE /api/cfo/conversations/[id]` - Delete conversation

4. **Web UI** (`src/app/(dashboard)/cfo/`):

## Migration Strategy

**`prisma/migrations/YYYYMMDDHHMMSS_cfo_agent/migration.sql`:**
```sql
CREATE TYPE "ChatSurface" AS ENUM ('WEB', 'SLACK');
CREATE TYPE "ChatMode" AS ENUM ('INTERNAL_CFO', 'PROPOSAL_BIZDEV', 'BOARD_INVESTOR');
CREATE TYPE "MessageRole" AS ENUM ('USER', 'ASSISTANT', 'TOOL', 'SYSTEM');

CREATE TABLE "Conversation" (
  "id" TEXT NOT NULL PRIMARY KEY,
  "companyId" TEXT NOT NULL,
  "userId" TEXT NOT NULL,
  "surface" "ChatSurface" NOT NULL,
  "mode" "ChatMode" NOT NULL DEFAULT 'INTERNAL_CFO',
  "slackChannel" TEXT,
  "slackThreadTs" TEXT,
  "title" TEXT,
  "summary" TEXT,
  "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
  "updatedAt" TIMESTAMP(3) NOT NULL,
  
  CONSTRAINT "Conversation_companyId_fkey" FOREIGN KEY ("companyId") REFERENCES "Company"("id"),
  CONSTRAINT "Conversation_userId_fkey" FOREIGN KEY ("userId") REFERENCES "User"("id")
);

CREATE INDEX "Conversation_companyId_userId_updatedAt_idx" ON "Conversation"("companyId", "userId", "updatedAt");
CREATE INDEX "Conversation_slackChannel_slackThreadTs_idx" ON "Conversation"("slackChannel", "slackThreadTs");

CREATE TABLE "Message" (
  "id" TEXT NOT NULL PRIMARY KEY,
  "conversationId" TEXT NOT NULL,
  "role" "MessageRole" NOT NULL,
  "content" JSONB NOT NULL,
  "modeAtTurn" "ChatMode" NOT NULL,
  "redactionEvents" JSONB,
  "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,
  
  CONSTRAINT "Message_conversationId_fkey" FOREIGN KEY ("conversationId") REFERENCES "Conversation"("id") ON DELETE CASCADE
);

CREATE INDEX "Message_conversationId_createdAt_idx" ON "Message"("conversationId", "createdAt");

CREATE TABLE "PersonaConfig" (
  "id" TEXT NOT NULL PRIMARY KEY,
  "companyId" TEXT NOT NULL UNIQUE,
  "voiceOverrides" JSONB,
  "toolOverrides" JSONB,
  "updatedAt" TIMESTAMP(3) NOT NULL,
  
  CONSTRAINT "PersonaConfig_companyId_fkey" FOREIGN KEY ("companyId") REFERENCES "Company"("id")
);
```

## Implementation Notes

### On Next.js 16 Async Params

Route handlers with `[id]` must use:
```ts
export async function GET(
  req: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params;
  // ...
}
```

### On Tool Calling

Following Anthropic SDK docs:
1. Build messages array from persisted conversation
2. Call `messages.create({ model: "claude-sonnet-4-20250514", system, tools, messages })`
3. If response has `tool_use` blocks, execute tools, append `tool_result`, loop
4. When `stop_reason === "end_turn"`, persist and return

### On Voice/Persona

From issue:
- "Plain English. No consulting jargon."
- "Numbers first, framing second."
- "Never uses emoji unprompted."
- "Dry, occasionally pointed."
- Avoid: "Let me know if you have any other questions!", "Based on my analysis..."

### On Existing Code Reuse

- **DO NOT reimplement** cost-basis calculation, project profitability, period comparison
- **DO** wrap existing functions in tool handlers
- **DO** reuse existing `prompt-builder.ts` patterns for narrative generation
