# Ticket: AI-generated financial summaries

**Issue:** https://github.com/artisgarbage/cl303-ledger-clone-chavez/issues/7
**Budget:** $50 USD

## Goal

Implement complete AI narrative generation pipeline for financial summaries:
1. Seed codelab303 P&L data from 4 XLSX files in setup_data/
2. Build generation API endpoints (on-demand + scheduled)
3. Create UI for on-demand generation and viewing
4. Support all narrative types (monthly, quarterly, annual, custom)

## Approach

### Phase 1: Dependencies & XLSX Parsing (High Priority)

1. **Install dependencies:**
   - `xlsx` for XLSX parsing
   - `@anthropic-ai/sdk` for AI generation (check if already installed)
   - `bcryptjs` appears to be missing from package.json but used in seed - add it
   - `zod` for validation (standard per stack)

2. **Create XLSX parser** (`prisma/lib/xlsx-parser.ts`):
   - Parse QuickBooks P&L format
   - Extract revenue, COGS, OpEx categories
   - Handle depth/indentation for line items
   - Skip subtotal rows (`isTotal: true`)

3. **Seed script** (`prisma/seed-financials.ts`):
   - Upsert FinancialPeriod + LineItem records
   - Handle all 4 files (2024 cash, 2025 cash, 2026 YTD accrual, 2026 YTD cash)
   - Idempotent - use composite unique keys
   - Integrate into main `seed.ts`

### Phase 2: Generation Engine

4. **Narrative builder** (`src/lib/narrative-builder.ts`):
   - Build structured prompt from financial data
   - Call Anthropic Claude API
   - Create dataSnapshot JSON for reproducibility
   - Return formatted narrative

5. **Environment variables:**
   - Create `.env.local.example` with:
     - `ANTHROPIC_API_KEY`
     - `CRON_SECRET`
     - `DATABASE_URL` (already implied)

### Phase 3: API Routes

6. **POST /api/narratives/generate:**
   - Admin auth check
   - Zod validation for request body
   - Call narrative builder
   - Store result in DB
   - Return narrative

7. **GET /api/narratives/scheduled:**
   - CRON_SECRET header validation
   - Date-based cadence logic (monthly/quarterly/annual)
   - Idempotent - check existing narratives
   - Generate missing summaries

8. **GET /api/narratives & GET /api/narratives/[id]:**
   - List endpoint with pagination
   - Single fetch endpoint
   - Both admin-only

### Phase 4: UI

9. **Narratives page** (`src/app/(dashboard)/narratives/page.tsx`):
   - Form with type dropdown, date pickers
   - Generate button with loading state
   - Display generated narrative (markdown → HTML)
   - List 10 most recent narratives
   - View/expand functionality

### Phase 5: Testing

10. **Vitest tests:**
    - API route tests following pattern in `src/app/api/status/route.test.ts`
    - XLSX parser tests with sample data
    - Narrative builder unit tests (mock Anthropic)
    - Integration test: seed → generate → fetch
