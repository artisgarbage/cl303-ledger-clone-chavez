# Ticket: Add GET /api/healthz endpoint

## Goal

Add an unauthenticated `GET /api/healthz` endpoint for Docker health checks, load balancer probes, and uptime monitors. The endpoint must return structured health status including database reachability, app version, and uptime.

## Approach

Implementation follows the exact specification from the GitHub issue:

1. **Route Handler** (`src/app/api/healthz/route.ts`):
   - Return 200 OK with structured JSON when healthy
   - Return 503 Service Unavailable when DB is unreachable
   - Include: status, timestamp, uptime, db status, version
   - Use `SELECT 1` for DB ping via Prisma
   - Never cache (force-dynamic)

2. **Middleware** (`src/middleware.ts`):
   - Add `/api/healthz` to public paths allowlist
   - Ensure no authentication required

3. **Dockerfile**:
   - Add HEALTHCHECK instruction using curl
   - Install curl in runner stage

4. **Tests** (`src/app/api/healthz/route.test.ts`):
   - 8 test cases covering healthy and degraded states
   - Mock Prisma for unit testing
   - Verify error handling doesn't leak internal errors
