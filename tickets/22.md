# Ticket 22 — TIK-013 Milestone 1: Billing Primitives

## Context

This is **Milestone 1** of the 7-milestone epic TIK-013. We're implementing the pricing, entitlements, and authorization system for Margot's two-rail pricing model (Rail A: human agency owners, Rail B: AI agents calling Margot).

Milestone 1 is **backend-only**: no UI changes, no Stripe integration, no agent endpoints. Just the data model, entitlement logic, and hooks into existing generation flows.

## Reconnaissance Findings

### Existing Codebase State
- **Next.js 16**, React 19, Prisma 7 on PostgreSQL, NextAuth v5
- **Billing tables already exist** in schema: `Plan`, `Subscription`, `UsageEvent`, `OverageCharge`, `AgentIdentity` (added in previous M1 PR #23)
- **Learnings indicate PR #23 was merged as commit `a4b8f4c`** but post-merge fixes were needed for Prisma import paths and optional chaining
- **Current test status**: 267 tests passing per learnings
- **Existing auth pattern**: `requireSession()` helper in `src/lib/auth-helpers.ts`
- **Existing audit logging**: `AccessAudit` table and `logAccess()` in `src/lib/audit.ts`
- **Narrative generation**: `/api/narratives/generate/route.ts`
- **CFO chat**: `/api/cfo/chat/route.ts`

### Stack Verification
- ✅ Prisma 7 (`@prisma/client: ^7.8.0`)
- ✅ Next.js 16 (`next: 16.2.4`)
- ✅ Zod 4 (`zod: ^4.4.3`)
- ✅ date-fns (`date-fns: ^4.1.0`)
- ✅ Vitest configured (`vitest.config.ts`)

### Key Constraints from Learnings
1. **Prisma instantiation pattern**: Must use `@prisma/adapter-pg` with connection string
2. **Prisma imports**: Use `@/lib/prisma` (NOT `@/lib/db`)
3. **Optional chaining**: Always use `entitlements.overage?.narrative` for nullable fields
4. **Date handling**: Use 24h windows (`gte/lt`) for date queries, not exact timestamps
5. **No npm install in sandbox**: Tests written but not executed locally

## Goal

Implement complete billing metering substrate with entitlement enforcement hooked into existing generation endpoints.

## Approach

### 1. Plan Definitions & Seed Script

**File**: `src/lib/billing/plans.ts`
- Define `Entitlements` TypeScript type
- Export `PLAN_DEFINITIONS: Record<PlanSlug, { ... }>` with all 9 plans
- Source all values from `strategy/PRICING_AND_GTM.md` §4

**File**: `prisma/seed-plans.ts`
- Upsert all 9 plan definitions (idempotent by slug)
- Wire into existing `prisma/seed.ts`

### 2. Entitlement Resolution Library

**File**: `src/lib/billing/entitlements.ts`
- `getActivePlan(companyId)` — returns FREE plan with synthetic subscription if none exists
- `recordUsage(opts)` — writes UsageEvent, returns running total + overage info
- `assertEntitlement(companyId, key)` — throws typed error if missing
- `checkQuota(companyId, kind)` — pure read, returns usage stats
- `computeOverage(companyId, periodStart, periodEnd)` — pure function

Use React `cache()` for request-scoped memoization of `getActivePlan()`.

### 3. Typed Errors

**File**: `src/lib/billing/errors.ts`
- `PlanUpgradeRequired` — extends Error with `requiredPlanSlug` and `reason`
- `QuotaExceeded` — extends Error with `kind`, `used`, `cap`, `overageAvailable`
- `EntitlementDenied` — generic fallback

### 4. Hook Into Existing Endpoints

**Modify**: `src/app/api/narratives/generate/route.ts`
- Add entitlement check before generation
- Record `NARRATIVE_GENERATED` usage AFTER successful generation
- Return 402 with upgrade payload on Free plan quota exhaustion
- Return 200 with overage on paid plans

**Modify**: `src/app/api/cfo/chat/route.ts`
- Check mode entitlement (Proposal/Board require Studio+)
- Record `CFO_TURN` + mode-specific usage AFTER successful response
- Return 402/403 with structured error

### 5. Agent Scope Strings

**File**: `src/lib/billing/agent-scopes.ts`
- Define `AGENT_SCOPES` const array (for M6, but define now)

### 6. Tests

**File**: `src/lib/billing/entitlements.test.ts`
- Plan resolution for each slug
- FREE fallback when no subscription exists
- Quota math across period boundaries
- Mode gating logic

**File**: `src/lib/billing/overage.test.ts`
- Under-cap returns 0 overage
- At-cap returns 0 overage
- Over-cap computes correct cents

**File**: `src/lib/billing/plans.test.ts`
- Snapshot test of `PLAN_DEFINITIONS` (price drift detection)

**Integration**: Extend `src/app/api/cfo/conversation-flow.test.ts`
- FREE hitting narrative cap → 402
- STARTER blocked from Proposal mode → 402
- STUDIO allowed for all modes → 200 with overage tracking
